tldraw Homelab Deployment Proposal

Option B: proper self-hosted collaborative board with auth + projects

Important finding: tldraw.dev/quick-start is the SDK. The official tldraw sync engine provides real-time rooms and asset storage, but does not ship authentication, authorization, projects, or a room registry. This proposal adds those as a custom layer on top of tldraw sync, deployed via GitOps on k3s.

What tldraw sync gives us

  • • Real-time WebSocket rooms (TLSocketRoom)
  • • Client sync via useSync / @tldraw/sync
  • • In-memory or SQLite per-room persistence
  • • Asset upload/download hooks (TLAssetStore)
  • • Bookmark unfurling hooks

What we have to build ourselves

  • • User authentication (OIDC via Authentik)
  • • Project/room registry and permissions
  • • Long-term snapshots + history
  • • Room listing, search, sharing
  • • Rate limits, upload size limits

Homelab fit

  • • Live DB on local-path (fast), snapshots to OMV MinIO
  • • Assets served from OMV MinIO S3 bucket
  • • Public ingress via Cloudflare Tunnel
  • • SOPS-encrypted secrets in public repo
  • • Logs/metrics/traces via OTel + Grafana

Open decisions

  • • Build custom backend or start from foxxmd/tldraw-selfhosted?
  • • Authentik vs oauth2-proxy as OIDC gate?
  • • Single replica (SQLite) or shared-state for horizontal scale?
  • • tldraw hobby/commercial license for non-localhost domain
  • • Room/project data model and UI for project switcher
